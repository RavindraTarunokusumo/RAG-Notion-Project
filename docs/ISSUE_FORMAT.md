# Issue Formatting Standard

Use this format for all project issues.

## Title

`NRAG-<ID>: <Clear action-oriented summary>`

Example: `NRAG-071: Streamlit auto-save on prompt submit and response completion`

## Body Template

```md
## Summary
One short paragraph describing the user/business outcome.

## Scope (<branch-or-epic>)
- Concrete implementation bullets (what is included).

## Acceptance Criteria
- [ ] Testable requirement 1
- [ ] Testable requirement 2

## References
- `path/to/file.py`
- `path/to/other_file.md`
```

## Style Rules

- Keep `Summary` to 1-3 sentences.
- Keep `Scope` focused on included work only.
- Put only verifiable, testable checks in `Acceptance Criteria`.
- Use repository file paths in `References`.
- Use `enhancement` label for feature requests unless another label is clearly more appropriate.
